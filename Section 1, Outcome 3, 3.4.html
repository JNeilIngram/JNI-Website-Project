3.4 Describe how you would go about testing digital solutions

Testing a digital solution comes down to examining all possible outcomes and seeing if there are any weaknesses.

In this project it comes down to examining every page and every navigation item to ensure that there are no "dead ends" and all the pages display correctly.

For more complex projects, the testing would tailored to project, but essentially test for the same things, namely :-

  1. Does it work?
  2. Can it be "crashed", and if so where and how?
  3. Can it be used for its intended purpose (not as crazy as it sounds)?
  4. Does it meet the project brief (again, not as crazy as it sounds)?

As discussed on other pages, without careful management projects can go seriously off track and deliver systems that were not requested!
