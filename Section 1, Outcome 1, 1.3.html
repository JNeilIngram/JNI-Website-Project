Planning starts with consultations with the client in order to produce an initial brief for the project. Items that need to be agreed at this point :-

  1. The specifics of the task
  2. Provisional budget
  3. Provisional timescale
  4. Survey the available hardware and infrastructure
  5. Access to the potential users (if it is an internal task)

At this point an initial project plan can be drawn up using kanban pinciles on a site such as trello. This is essentially a "to do" list, which can be accessed and updated by all participants in the project.

With the initial project plan drafted, further consulations with the client need to be carried out to finalise the budget, timescale and finalise the brief, with consideration as to what is possible and what the client expects.

The "final" project plan can then be produced an work can begin. Note the quotes round th final: no project plan can be set in stone, if you are not flexible you cannot react to unexpected problems and/or changes.
